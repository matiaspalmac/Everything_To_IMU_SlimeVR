# Contributing

Thanks for considering a contribution. This is a personal fork that diverges substantially from the upstream `Sebane1/Everything_To_IMU_SlimeVR` project — please read this before opening a PR so neither side wastes time.

## Before you write code

- **Open an issue first** for anything beyond a typo / docs fix / one-line bug fix. Even small features can clash with in-flight work or with the project's scope. A two-line issue describing what you want to do is enough.
- **The fork's scope** is the bridge layer: read IMU from controllers, fuse, hand a tracker to SlimeVR Server. Calibration, body-part assignment, reset / mounting, and skeleton are owned by SlimeVR Server itself — patches that re-implement those will be declined.
- **Hardware-specific changes** (DualSense, Joy-Con 2, Switch Pro, Wii, 3DS) need to be testable on the matching hardware. If you don't have the device, say so in the PR — the fix may need to wait until somebody can validate it physically.

## Getting set up

Requirements:
- Windows 11 (build 22000+).
- .NET 10 SDK.
- A clone of the repo with submodules: `git clone --recurse-submodules`.

Build:

```bash
dotnet build Everything-To-IMU-SlimeVR.sln -c Debug
```

Run from the WPF UI project (`Everything-To-IMU-SlimeVR.UI.WPF`). The single-file release recipe is in the README.

## Coding standards

- **C# 12 / .NET 10**. Follow the existing style — there is no formal `.editorconfig` per file, but match the surrounding code.
- **Comments explain *why*, not *what*.** A non-obvious hardware quirk, a subtle race, a workaround for a specific bug — those go in comments. "Increment counter" does not.
- **No AI attribution** in commits, PRs, or code. No `Co-Authored-By: Claude`, no `Generated by`, no shoutouts. The author is the person hitting the keyboard.
- **No upstream PRs from this fork.** Push to `origin` only. If you want a change to land upstream, send it directly to `Sebane1/Everything_To_IMU_SlimeVR`.
- **Submodule discipline.** `SlimeImuProtocol/` is its own repo; commit there first, then bump the pointer in the main repo as a separate commit.

## Commit messages

Conventional Commits, scoped to the area touched:

```
feat(joycon2): mag hard-iron calibration + reset buttons

Two features behind 9D fusion and manual reset gestures.

Magnetometer calibration
  * Factory hard-iron bias read from flash 0x13100 ...
```

- Subject line ≤ 72 characters, imperative mood (`fix(...)`, not `fixed(...)`).
- Body explains the reasoning. Wrap at 80 cols.
- Reference issue numbers when relevant. No GitHub keywords (`closes #`) unless the PR really should close the issue on merge.

## Pull request checklist

Before requesting review:

- [ ] `dotnet build Everything-To-IMU-SlimeVR.sln -c Release` produces 0 errors and 0 warnings.
- [ ] If you touched `SlimeImuProtocol/`, the submodule commit is pushed and the main repo points at it.
- [ ] If you touched a hardware path, you have tested with the hardware on hand and said so explicitly.
- [ ] No third-party native binaries added (vqf.dll / JoyShockLibrary.dll come from upstream and ship pre-built).
- [ ] No telemetry, no analytics, no remote-config endpoints. The bridge is local-only by design.

## Reporting a security issue

See [SECURITY.md](SECURITY.md) — please do **not** file a public issue.
